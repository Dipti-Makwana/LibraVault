What it does
Key Features

Every feature is backed by real SQL — joins, constraints, and transactions — designed to be Oracle-internship worthy.

📚

Book Management (CRUD)

Add, edit, delete, and search books by title, author, genre, or ISBN using optimized Oracle SQL queries.

👥

Member Registration

Register library members, track join dates, email contacts, and enforce unique constraints on email fields.

🔄

Borrow & Return

Track borrow/return dates automatically. Books update availability status using COMMIT/ROLLBACK transactions.

🔗

SQL JOIN Reports

Multi-table JOIN queries to generate reports like "all books currently borrowed by a specific member."

🛡

Data Integrity

Foreign key constraints, primary keys, and VARCHAR2 types following Oracle-specific best practices.

⚠️

Error Handling

Graceful Python exceptions for "book not found," "member at borrow limit," and DB connection failures.

📊 SQL JOIN Report

-- Books currently borrowed with member info
SELECT m.name, m.email, b.title, b.author, t.borrow_date, t.return_date
FROM Transactions t
JOIN Books b ON t.book_id = b.book_id
JOIN Members m ON t.member_id = m.member_id
WHERE b.is_available = 'N'
ORDER BY t.borrow_date DESC;

Getting Started
Setup Guide

Get the project running locally in four steps.

1

Clone the repository

Download the source code to your local machine.

git clone https://github.com/Dipti-Makwana/LibraVault
cd library-management-system
2

Install the Oracle Python driver

Uses Oracle's official modern thin driver — no Oracle Client installation needed.

pip install oracledb
3

Create the database schema

Run the provided SQL file in SQL*Plus or SQL Developer to create all tables and seed data.

@database/schema.sql
4

Configure and run

Update config.py with your Oracle credentials, then launch the app.

python app.py
